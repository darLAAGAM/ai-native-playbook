# Paywall Upgrade CRO

## Purpose

Improve in-product paywalls, upgrade prompts, and feature gates.

## Use When

Use for freemium conversion, trial expiration, usage limits, tier upgrades, premium feature gates, or upsell modals.

## Inputs

- Business model and plans
- Current upgrade triggers
- Feature or usage limits
- Conversion rate
- User value moments
- Pricing and checkout flow

## Procedure

1. Identify whether the prompt appears before, during, or after the user has experienced value.
2. Classify trigger type: feature gate, usage limit, trial expiration, time-based prompt, or behavior signal.
3. Clarify what the user gains, what is currently blocked, and the next best alternative.
4. Design copy around value and context, not just plan names.
5. Provide a clear upgrade path and a respectful escape path.
6. Define frequency caps and cool-down rules.
7. Measure paywall impressions, click-through, upgrade completion, churn, and revenue per user.

## Output

- Paywall or upgrade prompt design
- Trigger rules
- Copy and CTA
- Checkout friction notes
- Metrics and test plan

## Guardrails

- Do not block critical user work without a clear explanation.
- Do not surprise users with hidden limits.
- Do not use dark patterns around cancellation, trial terms, or plan selection.

## How to adapt

Customize trigger thresholds, plan names, usage limits, checkout requirements, and your definition of the product's value moment.
