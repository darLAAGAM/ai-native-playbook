# Onboarding CRO

## Purpose

Help new users reach first value faster and improve activation.

## Use When

Use for post-signup onboarding, first-run experience, empty states, activation funnels, setup checklists, or stalled users.

## Inputs

- Product or service context
- Activation event
- Current onboarding steps
- Drop-off by step
- User segments
- Email or in-app messaging

## Procedure

1. Define the earliest measurable action that predicts retention or repeat purchase.
2. Map the path from signup to that action.
3. Remove, defer, or simplify steps that do not directly support first value.
4. Design one clear goal for the first session.
5. Improve empty states, checklists, tooltips, and success states.
6. Add behavior-triggered recovery for stalled users.
7. Define activation, time-to-value, and retention metrics.

## Output

- Onboarding audit
- Recommended first-session flow
- Checklist or empty-state copy
- Recovery triggers
- Metrics plan

## Guardrails

- Do not teach every feature in the first session.
- Do not block core value behind unnecessary setup.
- Do not use generic tours if users need task-specific guidance.

## How to adapt

Define your activation event, key user segments, support escalation points, and lifecycle messaging rules.
