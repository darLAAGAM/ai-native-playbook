# Postmortem Or Retrospective

## Purpose

Convert a failed launch, incident, or project into learnings and prevention actions.

## Use When

Use when you need to convert a failed launch, incident, or project into learnings and prevention actions.

## Inputs

- Timeline
- Metrics impact
- People/process involved
- Customer impact
- Existing docs

## Procedure

1. Build a factual timeline from first signal to resolution.
2. State impact in customer, revenue, cost, and team terms.
3. Identify root causes using “why” analysis without blaming individuals.
4. Separate prevention, detection, response, and communication improvements.
5. Assign actions with owner, deadline, and verification method.

## Output

- Timeline
- Impact
- Root causes
- Action list
- Learning summary

## Guardrails

- No blame language.
- Do not call something “root cause” if it is only a symptom.

## How to adapt

Customize severity, review attendees, and where learnings are stored.
