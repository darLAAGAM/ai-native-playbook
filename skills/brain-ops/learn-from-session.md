# Learn From Session

## Purpose

Capture durable learnings from completed work so future agents do not rediscover the same thing.

## Use When

Use this skill when an operator needs a repeatable procedure for capture durable learnings from completed work so future agents do not rediscover the same thing.

## Inputs

- Conversation/session notes
- Changed files or decisions
- Tool issues
- User feedback

## Procedure

1. Identify only durable learnings: reproducible gotchas, decisions, preferences, working patterns, or failure patterns.
2. Discard trivial mistakes, one-off details, and unverified guesses.
3. Classify each learning: tool gotcha, process pattern, personal preference, business decision, or agent behavior.
4. Write each learning to the right knowledge location with date, source, and confidence.
5. If the learning is sensitive, store it privately or summarize safely.

## Output

- Learning entries
- Destination paths
- Confidence
- Open questions

## Guardrails

- Do not store secrets or unnecessary personal data.
- Do not overwrite prior knowledge without preserving history.

## How to adapt

Map categories to your own brain directory structure and privacy policy.
