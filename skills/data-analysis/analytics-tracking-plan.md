# Analytics Tracking Plan

## Purpose

Define the events, properties, conversions, and validation steps needed for decision-grade analytics.

## Use When

Use before implementing analytics, auditing tracking, launching a new funnel, or fixing unreliable reporting.

## Inputs

- Business questions
- Funnel or journey map
- Current analytics tools
- Key conversion actions
- Privacy or consent requirements
- Implementation owner

## Procedure

1. Start with decisions the data must inform.
2. Map the journey and define key events for each stage.
3. Choose event names and properties using one consistent convention.
4. Mark conversion events and explain counting rules.
5. Define user, session, campaign, and product properties only where useful.
6. Create a validation checklist for duplicates, missing properties, mobile, and consent.
7. Document ownership and update cadence.

## Output

- Tracking plan table
- Event and property dictionary
- Conversion definitions
- UTM or campaign conventions if needed
- QA checklist

## Guardrails

- Do not track everything just because it is possible.
- Do not collect personal data unless policy and law allow it.
- Do not ship tracking without a validation plan.

## How to adapt

Customize naming conventions, privacy rules, data retention, consent mode, and the tools used for implementation.
