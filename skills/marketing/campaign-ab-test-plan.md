# Campaign A/B Test Plan

## Purpose

Design a statistically credible paid campaign test before spend is committed.

## Use When

Use when testing ad copy, creative, audiences, landing pages, bids, or campaign structures.

## Inputs

- Test hypothesis
- Control and variant descriptions
- Baseline metric and conversion rate
- Minimum detectable effect
- Daily traffic or impressions
- Primary, secondary, and guardrail metrics

## Procedure

1. Rewrite the idea as a falsifiable hypothesis: because X, changing Y should improve Z for audience A.
2. Define one primary metric and the minimum lift worth acting on.
3. Calculate required sample size and expected duration before launch.
4. Confirm the test changes only the intended variable, or document all changed variables.
5. Set traffic allocation, dates, exclusion rules, and stop conditions.
6. Create a monitoring cadence for data quality, spend, tracking, and guardrail metrics.
7. Decide in advance how to call the result: winner, loser, inconclusive, or rerun.

## Output

- Test plan
- Sample size and duration estimate
- Metric definitions
- Launch checklist
- Decision rules

## Guardrails

- Do not call a winner before sample size or duration requirements are met.
- Do not optimize on vanity metrics if the business metric declines.
- Do not run overlapping tests on the same audience unless contamination is controlled.

## How to adapt

Customize confidence thresholds, acceptable test duration, platform constraints, and the metrics your business uses for profitability.
