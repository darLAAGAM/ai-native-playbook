# Onboarding Plan

## Purpose

Create a practical 30/60/90-day onboarding plan for a new hire.

## Use When

Use this skill when an operator needs a repeatable procedure for create a practical 30/60/90-day onboarding plan for a new hire.

## Inputs

- Role description
- Manager expectations
- Company docs
- Tools access checklist
- Success metrics

## Procedure

1. Define role outcomes for day 30, 60, and 90.
2. List required access, training, recurring meetings, and reference docs.
3. Create a first-week schedule with people to meet and tasks to complete.
4. Define early deliverables that prove understanding without creating unnecessary risk.
5. Add manager check-in questions and evaluation criteria.

## Output

- First-week plan
- 30/60/90 outcomes
- Access checklist
- Manager check-in prompts

## Guardrails

- Do not overload day one with every system.
- Do not assign risky production work before context and access are ready.

## How to adapt

Customize by role, seniority, remote/hybrid setup, and compliance needs.
