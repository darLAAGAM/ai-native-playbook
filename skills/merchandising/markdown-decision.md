# Markdown Decision

## Purpose

Decide when and how much to discount slow-moving inventory without damaging margin unnecessarily.

## Use When

Use this skill when an operator needs a repeatable procedure for decide when and how much to discount slow-moving inventory without damaging margin unnecessarily.

## Inputs

- SKU/category sales history
- Current inventory
- Gross margin
- Seasonality
- Promo calendar

## Procedure

1. Identify aged or slow stock using launch age, sell-through, weeks of cover, and velocity trend.
2. Separate product problem from visibility problem: stock depth, channel placement, content, or display.
3. Model markdown scenarios and expected gross margin impact.
4. Choose action: no markdown, improve visibility, bundle, targeted discount, public markdown, or liquidation.
5. Set review date and success metric.

## Output

- Markdown recommendation
- Rationale
- Margin impact
- Execution channel
- Review metric

## Guardrails

- Do not discount before fixing obvious visibility or allocation issues.
- Do not ignore brand/margin impact of public discounts.

## How to adapt

Set markdown thresholds by category, season, and margin floor.
